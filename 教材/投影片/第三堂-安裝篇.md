## Slide 3: 常見問題排解

![問題排解](./images/unified_14_troubleshoot_1767752381900.png)

**標題**：問題不可怕，有解法！

---

### 📝 講師備忘錄

**⚠️ 這是本堂課最重要的部分！（20 分鐘）**

現場帶學員操作，遇到問題即時排解：

---

**問題 1：PowerShell 執行權限（最常見）**

> 「幾乎每個人第一次都會遇到這個。Windows 預設不讓你執行腳本。」

解法（帶學員一起輸入）：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

然後輸入 `Y` 確認。

---

**問題 2：Port 8964 被佔用**

> 「如果之前開過 Revit MCP 沒有正常關閉，Port 可能還被佔著。」

診斷：
```powershell
netstat -ano | findstr :8964
```

解法：
```powershell
# 記下 PID 數字，然後
taskkill /PID [那個數字] /F
```

---

**問題 3：編譯找不到 RevitAPI**

> 「這通常是 Revit 安裝路徑不標準造成的。」

解法：
1. 開啟 .csproj 檔案
2. 找到 RevitAPI 的 HintPath
3. 改成你電腦上的實際路徑

---

**問題 4：DLL 複製到錯誤位置**

正確路徑公式：
```
%APPDATA%\Autodesk\Revit\Addins\{版本號}\RevitMCP\
```

例如：
```
C:\Users\你的名字\AppData\Roaming\Autodesk\Revit\Addins\2024\RevitMCP\
```

## Slide 4a: 方案 A - Google Antigravity

**標題**：工具與知識庫的管理核心

---

### 📝 講師備忘錄

**Antigravity 設定（10 分鐘）：**

> 「Antigravity 是 Google 的 AI 開發環境，非常適合用來管理 domain/ 目錄和建立長期工作流程。」

**設定步驟**：
1. 開啟終端機，編輯 `~/.gemini/settings.json`
2. 加入 mcpServers 區塊
3. 填入正確的絕對路徑

**設定範例**：
```json
{
  "mcpServers": {
    "revit-mcp": {
      "command": "node",
      "args": ["C:\\你的路徑\\MCP-Server\\build\\index.js"]
    }
  }
}
```

**⚠️ 重要**：
- 路徑必須是**絕對路徑**
- 每次重開機後，確保先執行過 `npm run build`
- Port 8964 必須在 Revit 端啟動後才能連接

## Slide 4b: 方案 B - Gemini CLI

**標題**：終端機介面，直覺省資源

---

### 📝 講師備忘錄

**Gemini CLI 設定（10 分鐘）：**

> 「如果你習慣用終端機工作，Gemini CLI 是最佳選擇。它省資源、反應快，而且啟動時就自動掛載好 MCP Server。」

**日常啟動 SOP**：
1. 開啟終端機，進入 `MCP-Server` 目錄
2. 執行 `npm run build`（確保最新版本）
3. 開啟 Revit，點擊「MCP 服務（開/關）」
4. 在終端機輸入 `gemini` 啟動 CLI
5. 輸入 `/mcp list` 確認連接成功

**與 Revit 互動**：
直接在 CLI 輸入自然語言，例如「請列出所有樓層」

## Slide 4c: 方案 C - Claude Desktop

**標題**：初學者友善的入門方案

---

### 📝 講師備忘錄

**Claude Desktop 設定（5 分鐘）**：

> 「如果你是第一次接觸 MCP，Claude Desktop 是最簡單的選擇。」

設定只需要：
1. 開啟設定
2. 找到 MCP Servers
3. 加入 revit-mcp
4. 貼上路徑
5. 重啟

**設定範例**：
```json
{
  "revit-mcp": {
    "command": "node",
    "args": ["C:\\你的路徑\\MCP-Server\\build\\index.js"]
  }
}
```

**⚠️ 注意**：路徑要用**絕對路徑**，不能用相對路徑！

## Slide 5: 驗證安裝成功

**標題**：測試連線

---

### 📝 講師備忘錄

**最後驗證（5 分鐘）**：

**如果失敗**：
- 檢查 Revit 有沒有開啟 MCP 服務
- 檢查 Claude Desktop 有沒有重啟
- 檢查設定檔路徑是否正確
