## Slide 2: AI 協作指令

**標題**：三個魔法指令

---

### 📝 講師備忘錄

**詳細解說（15 分鐘）**：

---

**指令 1：`/lessons`**

> 「當一段對話很成功，你想把它記錄下來。」

使用方式：
```
你：/lessons
AI：我會從這段對話中提煉關鍵規則，
    並記錄到 GEMINI.md 的「智慧提煉」章節。
```

範例：
```
提煉結果：
[L-003] 消防栓標記策略
- 規則：消防栓查詢應使用族群名稱模糊匹配
- 實踐：篩選條件包含「消防栓」「Fire_Hydrant」「FH」
```

---

**指令 2：`/review`**

> 「當你懷疑文件太亂、資訊過時。」

使用方式：
```
你：/review
AI：我會檢查 GEMINI.md 和 domain/ 目錄，
    找出需要整理或遷移的內容。
```

---

**指令 3：`/explain`**

> 「當你需要理解複雜概念。」

使用方式：
```
你：/explain MCP 架構
AI：（用視覺化方式解釋，包含表格、流程圖）
```

> 「這個指令會**強制 AI 用圖表解釋**，避免一大堆文字。」

## Slide 4: domain 目錄結構

**標題**：知識的家

---

### 📝 講師備忘錄

**目錄結構解說（10 分鐘）**：

```
domain/
├── fire-rating-check.md      ← 防火檢查 SOP
├── floor-area-review.md      ← 容積檢討 SOP
├── corridor-analysis-protocol.md ← 走廊分析 SOP
├── element-coloring-workflow.md  ← 元素上色 SOP
├── qa-checklist.md           ← 品質檢查清單
└── references/
    └── building-code-tw.md   ← 台灣建築法規參考
```

---

**每個 .md 檔案的結構**：

```markdown
# 流程名稱

## 觸發條件
什麼時候該執行這個流程？

## 步驟
1. 使用工具 A 做什麼
2. 使用工具 B 做什麼
3. ...

## 範例對話
示範正確的對話方式

## 注意事項
可能遇到的問題和解決方法
```

## Slide 5: 版本控制與協作

**標題**：團隊共享知識

---

### 📝 講師備忘錄

**團隊協作概念（5 分鐘）**：

> 「domain/ 目錄可以放在 Git 儲存庫裡，這樣團隊可以共享：
> 
> - 成功的工作流程
> - 學到的經驗教訓
> - 公司內部的標準做法」

---

**Git 工作流程簡介**：

```
你完成一個好用的流程
    ↓
寫成 domain/your-workflow.md
    ↓
git add / git commit
    ↓
同事 git pull
    ↓
同事也能用這個流程！
```

---

**提醒**：

> 「這不是 Git 課程，但你要知道：這些知識檔案是可以像程式碼一樣版本控制的。
> 
> 如果團隊有專人維護這些 domain 檔案，整個團隊的效率都會提升。」
