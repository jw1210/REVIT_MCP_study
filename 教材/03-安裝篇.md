# 第二部分：安裝篇

## 環境設定完全指南

---

### 2.1 環境需求檢查清單

開始安裝前，請確認以下項目：

| 項目 | 需求 | 檢查方式 |
|:----|:----|:--------|
| **作業系統** | Windows 10 或更新 | 設定 → 系統 → 關於 |
| **Revit** | 2022 / 2023 / 2024 / 2025 / 2026 | 開啟 Revit → 幫助 → 關於 |
| **.NET Framework** | 4.8 (Revit 2022–2024) / .NET 8 (Revit 2025–2026) | 通常已內建 |
| **Node.js** | LTS 版本 (20.x+) | 命令列輸入 `node --version` |
| **硬碟空間** | 約 500MB | - |
| **網路連線** | 必需 | 用於 AI 通訊 |

### 2.3 步驟一：安裝 Node.js

> 💡 如果你已經安裝過 Node.js，可以跳過這步。

#### 檢查是否已安裝

1. 按 `Win + R`，輸入 `cmd`，按 Enter
2. 輸入：
   ```
   node --version
   ```
3. 如果顯示版本號（例如 `v20.10.0`），表示已安裝 ✅
4. 如果顯示「找不到命令」，請繼續以下步驟

#### 下載並安裝

1. 開啟瀏覽器，前往 **https://nodejs.org**
2. 點擊左邊的 **「LTS」** 按鈕（推薦版本）
3. 下載 Windows 安裝程式（`.msi` 檔案）
4. 執行安裝程式，一直點「Next」直到完成
5. **重新啟動電腦**

---

### 2.4 步驟二：編譯 Revit Add-in

#### 確認你的 Revit 版本

根據你的 Revit 版本，使用對應的專案檔：

| Revit 版本 | 建構組態 | 輸出路徑 | 備註 |
|:----------|:------|:--------|:--------|
| 2022 | `Release.R22` | `bin\Release\` | .NET Framework 4.8 |
| 2023 | `Release.R23` | `bin\Release\` | .NET Framework 4.8 |
| 2024 | `Release.R24` | `bin\Release\` | .NET Framework 4.8 |
| 2025 | `Release.R25` | `bin\Release\` | .NET 8, ElementId=long |
| 2026 | `Release.R26` | `bin\Release\` | .NET 8, ElementId=long |

> ℹ️ 專案已統一使用 `Nice3point.Revit.Sdk` 進行多版本建構，所有版本都使用同一個 `RevitMCP.csproj`。
> 舊的 `RevitMCP.2024.csproj` 已棄用，僅保留作為參考。

#### 編譯步驟

1. **開啟 PowerShell**
   - 按 `Win + X`，選擇「Windows PowerShell」

2. **進入專案目錄**
   ```powershell
   cd "C:\您的路徑\REVIT_MCP_study\MCP"
   ```

3. **執行編譯指令**
   
   根據您的 Revit 版本選擇對應的組態：
   ```powershell
   dotnet build -c Release.R22 RevitMCP.csproj   # Revit 2022
   dotnet build -c Release.R23 RevitMCP.csproj   # Revit 2023
   dotnet build -c Release.R24 RevitMCP.csproj   # Revit 2024
   dotnet build -c Release.R25 RevitMCP.csproj   # Revit 2025
   dotnet build -c Release.R26 RevitMCP.csproj   # Revit 2026
   ```

4. **確認成功**
   - 看到「Build succeeded」表示成功

#### 部署到 Revit

**方法 A：使用自動腳本（推薦）**

```powershell
cd ..\scripts
.\install-addon.ps1
```

> 💡 如果遇到權限問題，請先執行：
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```

**方法 B：手動複製**

1. 開啟 Revit Addins 資料夾：
   ```powershell
   explorer $env:APPDATA\Autodesk\Revit\Addins\2024
   ```
   （將 2024 改成你的版本號）

2. 建立 `RevitMCP` 資料夾

3. 複製以下檔案到該資料夾：
   - `RevitMCP.dll`（從 `bin\Release\`）
   - `RevitMCP.addin`（從 MCP 目錄）

### 2.5 步驟三：編譯 MCP Server

MCP Server 是 AI 和 Revit 之間的「翻譯官」。

1. **進入 MCP-Server 目錄**
   ```powershell
   cd "C:\您的路徑\REVIT_MCP_study\MCP-Server"
   ```

2. **安裝相依套件**
   ```powershell
   npm install
   ```
   - 等待 1-5 分鐘
   - 完成時看到「added XXX packages」

3. **編譯 TypeScript**
   ```powershell
   npm run build
   ```
   - 完成時會產生 `build/` 資料夾

---

### 2.6 步驟四：設定 AI 平台

選擇一個 AI 平台進行設定。建議初學者選擇 **Claude Desktop**。

---

#### 選項 A：Claude Desktop（推薦初學者）

1. **下載 Claude Desktop**
   - 前往 https://claude.ai/download
   - 點擊「Download for Windows」
   - 安裝並登入

2. **設定 MCP Server**
   - 開啟 Claude Desktop
   - 點擊右上角「⚙️ 設定」
   - 找到「MCP Servers」
   - 點擊「Add Server」

3. **填入資訊**
   ```
   名稱：revit-mcp
   命令：node
   參數：C:\您的路徑\REVIT_MCP_study\MCP-Server\build\index.js
   ```

4. **儲存並重啟 Claude Desktop**

---

#### 選項 B：Gemini CLI

1. **安裝 Gemini CLI**
   ```powershell
   npm install -g @google/gemini-cli
   ```

2. **編輯設定檔**
   - 按 `Win + R`，輸入 `%USERPROFILE%\.gemini`
   - 開啟 `settings.json`
   - 加入以下內容：
   
   ```json
   {
     "mcpServers": {
       "revit-mcp": {
         "command": "node",
         "args": ["C:\\您的路徑\\REVIT_MCP_study\\MCP-Server\\build\\index.js"]
       }
     }
   }
   ```

3. **驗證設定**
   ```powershell
   gemini
   /mcp list
   ```
   - 應該看到 `revit-mcp`

---

#### 選項 C：VS Code Copilot

如果你用 VS Code 開啟專案，設定已經準備好了！

1. **開啟專案**
   - 用 VS Code 開啟 `REVIT_MCP_study` 資料夾

2. **設定檔位置**
   - `.vscode/mcp.json` 已包含設定
   - 使用 `${workspaceFolder}` 變數，無需修改路徑

3. **使用 Copilot Chat**
   - 按 `Ctrl + Shift + I` 開啟聊天視窗

### 2.7 驗證安裝

#### 步驟 1：啟動 Revit

1. 開啟 Revit 2022/2023/2024
2. 載入或建立一個專案
3. 在「MCP Tools」面板，點擊「**MCP 服務 (開/關)**」

   > 看到「WebSocket 伺服器已啟動，監聽: localhost:8964」表示成功！

#### 步驟 2：測試 AI 連線

在你選擇的 AI 平台中輸入：

```
請列出這個 Revit 專案的所有樓層
```

如果 AI 回傳樓層清單，恭喜你，安裝成功！🎉

### 2.8 常見問題排解

#### ❌ 問題：「找不到 RevitMCP.dll」

| 原因 | 解決方法 |
|:----|:--------|
| 未執行編譯 | 執行 `dotnet build -c Release` |
| 版本不對應 | 2024 要用 `RevitMCP.2024.csproj` |
| 路徑錯誤 | 檢查是否在 MCP 目錄下 |

#### ❌ 問題：「Port 8964 已被佔用」

```powershell
# 檢查誰佔用了 8964
netstat -ano | findstr :8964

# 關閉佔用程式，或修改 Port
# 編輯 MCP-Server/src/index.ts 中的 PORT = 8964
```

#### ❌ 問題：「PowerShell 腳本執行錯誤」

```powershell
# 允許腳本執行
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

#### ❌ 問題：「MCP Server 連接失敗」

1. 確認 Revit 中已啟動 MCP 服務
2. 確認設定檔中使用**絕對路徑**
3. 確認已執行 `npm run build`
4. 重啟 AI 應用程式
